Тестирование страниц ошибок

На этой странице вы можете протестировать страницы ошибок приложения. Для корректной работы страниц ошибок необходимо установить DEBUG = False в файле settings.py.

Примечание
При DEBUG = True вместо пользовательских страниц ошибок Django будет показывать стандартные страницы отладки с подробной технической информацией.
404 - Страница не найдена
Ошибка возникает, когда запрашиваемая страница не существует.
Тестировать
500 - Ошибка сервера
Внутренняя ошибка сервера при обработке запроса.
Тестировать
403 - Доступ запрещен
Ошибка возникает при отсутствии прав доступа к ресурсу.
Тестировать
Внимание!
После тестирования рекомендуется вернуть значение DEBUG = True в файле settings.py для продолжения разработки.
